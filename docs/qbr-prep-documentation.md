## What It Does — Step by Step

```
  You type /qbr-prep
       │
       ▼
  Claude asks for inputs (partner, tier, quarter, region, data you have)
       │
       ▼
  SharePoint searched automatically for that partner
  (past QBRs, account docs, templates — GlobalPartnerEnablementNetwork)
       │
       ▼
  Full QBR generated using real UiPath KPI framework
       │
       ▼
  You choose an export format:
       ├── PowerPoint-ready (slide-by-slide bullet structure)
       ├── Word / exec doc
       ├── Pre-read email (to send partner before the meeting)
       ├── Action items only (clean task list)
       └── Save to SharePoint
```

## How to Use It

### Basic usage
```
/qbr-prep
```
Claude will ask you for all required inputs interactively.

### Pass context upfront (fastest)
```
/qbr-prep Partner: Accenture | Tier: Strategic | Quarter: Q2 FY27 | Region: EMEA
Pipeline: $2.3M sourced, $8.1M influenced
Certs: 12 new agentic, 45 total
Win: Heineken deal $420k ARR
Risk: Slow migration to Maestro
```

### Paste raw notes
```
/qbr-prep
[paste messy meeting notes, email threads, Salesforce data — anything raw]
```
Claude extracts structure from whatever you paste.

### Internal exec review (CFO / VP)
```
/qbr-prep
[describe: internal program review for CFO, use mock data]
```
Works for internal reviews too — not just partner-facing QBRs.

## Export Formats

After generating the QBR, Claude offers 5 formats:

| # | Format | Best for |
|---|--------|----------|
| 1 | **PowerPoint-ready** | Paste directly into slides — slide-by-slide structure |
| 2 | **Word / exec doc** | Clean formatted document to share |
| 3 | **Pre-read email** | Send to partner attendees before the meeting |
| 4 | **Action items only** | Drop into Teams, email, or project tracker |
| 5 | **Save to SharePoint** | Uploads to GlobalPartnerEnablementNetwork |

The `.pptx` file is generated by `exports/generate_qbr_pptx.py` using UiPath brand colours (dark navy + orange, RAG colour-coded tables).

## SharePoint Integration

When you provide a partner name, the skill automatically searches:
- `GlobalPartnerEnablementNetwork` — `# Programs/Partner Success/`
- `Team Internal/` — for strategic docs and past reviews
- Any folder containing the partner name

**Requires:** Microsoft 365 MCP connected in Claude Code  
**To connect:** Type `/mcp` in Claude Code → select "claude.ai Microsoft 365" → authenticate with your UiPath account

## How to Share With Your Team

### Option 1 — Project access (everyone on the team)
Anyone with access to the `ClaudeCode-PB/` project folder already has the skill.
The skill lives at: `.claude/commands/qbr-prep.md`

### Option 2 — Global install (works in any project)

**Mac / Linux** — run once in Terminal:
```bash
cp .claude/commands/qbr-prep.md ~/.claude/commands/qbr-prep.md
```

**Windows** — run once in PowerShell:
```powershell
Copy-Item ".claude\commands\qbr-prep.md" "$env:USERPROFILE\.claude\commands\qbr-prep.md"
```
> If the folder doesn't exist yet, create it first:
> ```powershell
> New-Item -ItemType Directory -Force "$env:USERPROFILE\.claude\commands"
> ```

After this, `/qbr-prep` works everywhere — not just in this project.

### Option 3 — Share the file directly
Send a teammate the file: `.claude/commands/qbr-prep.md`

- **Mac/Linux:** they drop it into `~/.claude/commands/`
- **Windows:** they drop it into `C:\Users\<their-username>\.claude\commands\`

They can do this via File Explorer — just paste the file into that folder (create it if it doesn't exist).

### What teammates need
1. Claude Code installed — [claude.ai/code](https://claude.ai/code)
2. Microsoft 365 MCP connected (`/mcp` → "claude.ai Microsoft 365")
3. Access to the `ClaudeCode-PB/` project folder (or the skill file)

## Customising the Skill

Edit `.claude/commands/qbr-prep.md` to tailor it to your needs:

| What to change | Where in the file | Example |
|---------------|------------------|---------|
| Add your name permanently | Replace `[Your name]` in Step 1 | `PSM: Mihai Gavan` |
| Add/remove KPI rows | Scorecard table in Step 3 | Add GSI-specific metrics |
| Change tone | Step 3 — Tone instruction | "More conversational for SMB partners" |
| Add a new section | Copy a section block in Step 3 | Add "Partner Investment ROI" |
| Change output format | Step 4 — post-generation options | Remove email, add Slack format |
| Pre-fill region | Step 1 required inputs | Default region: EMEA |

## Limitations & Known Issues

| Limitation | Workaround |
|------------|-----------|
| SharePoint MCP is read-only — cannot upload directly | Drag `.pptx` from `exports/` to SharePoint manually, or use OneDrive sync |
| Metrics are mock unless you paste real data | Connect Power BI agent (Track 3) for live data |
| PPTX generated via `python-pptx` — not editable in full brand template | Apply your UiPath master slide template after generation |
| Skill works best with specific partner data — generic input = generic output | The more data you paste, the better the output |
