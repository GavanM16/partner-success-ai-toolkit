Available Slide Layouts (10 Types)
coverTitle slide — always first
{ "type": "cover" }
// Uses top-level title, subtitle, presenter, date
agendaNumbered agenda items
{ "type": "agenda",
  "title": "Today's Agenda",
  "items": ["Section 1", "Section 2"] }
sectionFull-slide section divider
{ "type": "section",
  "number": "01",
  "title": "Section Title",
  "subtitle": "Optional sub" }
stat_row2–4 big metric boxes
{ "type": "stat_row", "title": "At a Glance",
  "stats": [
    { "value": "$2.6M", "label": "ARR", "color": "green" },
    { "value": "18",    "label": "Partners", "color": "orange" }
  ] }
bulletsTitle + bullet list (most common)
{ "type": "bullets",
  "title": "Slide Title",
  "subtitle": "Context line",
  "bullets": ["Point one", "Point two"] }
two_colSide-by-side content columns
{ "type": "two_col", "title": "Comparison",
  "left":  { "heading": "Option A", "bullets": [...] },
  "right": { "heading": "Option B", "bullets": [...] } }
tableData table — RAG colours auto-applied
{ "type": "table", "title": "KPI Scorecard",
  "headers": ["KPI", "Target", "Status"],
  "rows": [["NRR", "+10%", "🟢 Ahead"]] }
cards2–4 feature/info cards with icons
{ "type": "cards", "title": "Why Partner Success?",
  "cards": [
    { "icon": "🏆", "title": "Expert", "body": "..." }
  ] }
timelineHorizontal phase timeline
{ "type": "timeline", "title": "Roadmap",
  "steps": [
    { "phase": "Q1", "title": "Foundation",
      "dates": "Feb–Apr", "status": "done" }
  ] }
closingThank you / final slide
{ "type": "closing",
  "title": "Thank You",
  "message": "Questions & Discussion",
  "contact": "name@uipath.com" }
Share With Your Team
Mac / Linux

Run once in Terminal — downloads directly from GitHub, no project folder needed:

mkdir -p ~/.claude/commands
curl -o ~/.claude/commands/qbr-prep.md https://raw.githubusercontent.com/GavanM16/partner-success-ai-toolkit/main/.claude/commands/qbr-prep.md
curl -o ~/.claude/commands/deck-builder.md https://raw.githubusercontent.com/GavanM16/partner-success-ai-toolkit/main/.claude/commands/deck-builder.md
Installs both /qbr-prep and /deck-builder globally — works in any project.
Windows — PowerShell

Run once in PowerShell — downloads directly from GitHub, no project folder needed:

New-Item -ItemType Directory -Force "$env:USERPROFILE\.claude\commands"
Invoke-WebRequest -Uri "https://raw.githubusercontent.com/GavanM16/partner-success-ai-toolkit/main/.claude/commands/qbr-prep.md" -OutFile "$env:USERPROFILE\.claude\commands\qbr-prep.md"
Invoke-WebRequest -Uri "https://raw.githubusercontent.com/GavanM16/partner-success-ai-toolkit/main/.claude/commands/deck-builder.md" -OutFile "$env:USERPROFILE\.claude\commands\deck-builder.md"
No terminal? Use File Explorer — paste deck-builder.md into C:\Users\<you>\.claude\commands\ (create the folder if it doesn't exist).
Also needed: Copy agents/deck-builder/generate_deck.py to the same relative location so the PPTX engine can run. Full setup guide in README.md.
